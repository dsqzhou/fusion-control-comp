# 环境接口参考

本文档详细说明环境对外暴露的 observation、action、reference 和关键配置，面向在 standalone-env 进行训练与联调的选手。

**相关文档导航**：
- **赛题说明文档**：任务背景、评测规则和提交要求
- **README.md**：本地安装、仿真器启动、训练流程
- **submission/README.md**：推理服务镜像构建、HTTP 接口、约束条件（Dockerfile、start_infer.sh、run.sh 等不可修改）

## 1. 总体接口

环境采用 Gymnasium 风格接口：

- `obs, info = env.reset(seed=..., options=...)`
- `obs, reward, terminated, truncated, info = env.step(action)`
- `env.close()`

其中：

- `observation` 为全量字典
- `action` 为 12 维真实物理电压
- `reference` 会在每一步通过 observation 一并返回

## 2. 核心控制目标

比赛最核心的控制目标是 `Ip`、`R`、`Z` 和 `lcfs_points`。

| 字段 | 形状 | 含义 | 与控制目标的关系 |
| --- | --- | --- | --- |
| `Ip` | `(1,)` | 等离子体电流 | 直接对应电流跟踪目标 |
| `R` | `(1,)` | 等离子体水平位置 | 直接对应位置控制目标 |
| `Z` | `(1,)` | 等离子体垂直位置 | 直接对应位置控制目标 |
| `lcfs_points` | `(32, 2)` | LCFS 边界点序列，每个点为 `(R, Z)` | 直接对应位形/边界控制目标 |

说明：

- `lcfs_points[i] = [R_i, Z_i]`
- 可将 `lcfs_points` 理解为"边界轮廓采样点"

### `lcfs_points` 与底层环境变量的关系

底层环境中，LCFS 边界可由更原始的边界坐标数组表示：

- `rB`：边界点的 `R` 坐标，维度 `32`
- `zB`：边界点的 `Z` 坐标，维度 `32`

在 `environment` 中，既暴露 `rB` / `zB`，也统一封装为：

- `lcfs_points`：当前时刻边界点序列
- `reference_lcfs_points`：当前时刻目标边界点序列

`lcfs_points`更适合 AI 模型直接读取和奖励函数设计。边界点序列顺时针固定为 `(32, 2)`。

## 3. 其他 observation 字段

除核心目标外，环境还提供辅助状态量。这些量不是最终评分目标本身，但会影响对当前位形状态和动态趋势的判断。

### 3.1 线圈与整体形状相关

| 字段 | 形状 | 简要含义 | 作用 |
| --- | --- | --- | --- |
| `I_PF` | `(12,)` | PF 线圈电流 | 反映执行器当前状态，与动作响应直接相关 |
| `Rmax` | `(1,)` | 边界最大半径 | 辅助描述等离子体外轮廓 |
| `Rmin` | `(1,)` | 边界最小半径 | 辅助描述等离子体外轮廓 |
| `aminor` | `(1,)` | 小半径 | 辅助描述整体尺寸 |
| `deltal` | `(1,)` | 下三角形变 | 辅助描述下部形状变化 |
| `deltau` | `(1,)` | 上三角形变 | 辅助描述上部形状变化 |
| `kappa` | `(1,)` | 拉长比 | 辅助描述整体形状拉伸程度 |
| `rc` | `(1,)` | 几何中心半径 | 辅助定位整体位形中心 |
| `zc` | `(1,)` | 几何中心高度 | 辅助定位整体位形中心 |

### 3.2 磁通与磁诊断相关

| 字段 | 形状 | 简要含义 | 作用 |
| --- | --- | --- | --- |
| `FA` | `(1,)` | 磁轴处极向磁通 | 辅助反映磁平衡状态 |
| `FB` | `(1,)` | 边界点极向磁通 | 辅助反映边界相关磁状态 |
| `Bm` | `(100,)` | 磁探针磁场信号 | 提供边界和位形相关诊断信息 |
| `Fx` | `(4290,)` | 磁通网格展开向量 | 提供更细粒度的空间磁状态信息 |
| `rx` | `(66,)` | 磁通网格 `R` 方向坐标 | 与 `Fx` 配合使用 |
| `zx` | `(65,)` | 磁通网格 `Z` 方向坐标 | 与 `Fx` 配合使用 |

说明：

- `4290 = 66 x 65`
- `Fx` 可理解为二维磁通网格在 observation 中的展开表示

### 3.3 X 点相关

| 字段 | 形状 | 简要含义 | 作用 |
| --- | --- | --- | --- |
| `rX` | `(6,)` | X 点 `R` 坐标 | 辅助判断拓扑和偏滤器相关结构 |
| `zX` | `(6,)` | X 点 `Z` 坐标 | 辅助判断拓扑和偏滤器相关结构 |
| `nX` | `(1,)` | X 点数量 | 表示当前有效 X 点个数 |

### 3.4 状态标记

| 字段 | 形状 | 简要含义 |
| --- | --- | --- |
| `failure` | `(1,)` | 是否发生失败/破裂，取值为 `0` 或 `1` |

## 4. reference 字段

环境每一步 observation 都会带当前时刻 reference，便于直接构造跟踪控制器。

| 字段 | 形状 | 含义 |
| --- | --- | --- |
| `reference_Ip` | `(1,)` | 当前步目标电流 |
| `reference_R` | `(1,)` | 当前步目标水平位置 |
| `reference_Z` | `(1,)` | 当前步目标垂直位置 |
| `reference_lcfs_points` | `(32, 2)` | 当前步目标边界点序列 |

支持两类 reference 模式：

- `hold`：从 reset 初态自动生成整段 reference
- `trajectory`：由选手显式传入逐步 reference

当使用 `trajectory` 时，传入 `reset(options=...)` 的 reference 形状应为：

| 字段 | 期望形状 |
| --- | --- |
| `reference["Ip"]` | `(T,)` 或标量 |
| `reference["R"]` | `(T,)` 或标量 |
| `reference["Z"]` | `(T,)` 或标量 |
| `reference["lcfs_points"]` | `(T, 32, 2)` 或 `(32, 2)` |

其中 `T = max_steps`。

## 5. action 说明

动作空间固定为 12 维真实物理电压：

| 字段 | 形状 | 含义 |
| --- | --- | --- |
| `action` | `(12,)` | 12 维真实物理电压指令 |

**关键约束**：
- 训练阶段和提交阶段都必须输出 12 维动作
- 如果模型内部输出低维动作（例如 7 维），则需要在提交前自行映射回 12 维
- 环境默认动作上下界为：

```python
low  = [-1499, -230, -172, -172, -348, -348, -270, -270, -348, -348, -270, -270]
high = [ 1499,  230,  172,  172,  348,  348,  270,  270,  348,  348,  270,  270]
```

**7 维动作参考**（仅用于训练阶段快速实验）：
- 详见 `README.md` 中"7 维动作：经验降维参考"部分
- 映射逻辑：中间 10 维线圈按 5 组对称对处理，每组共享同一电压
- 不是比赛规则，仅供参考

## 6. reset 参数

`reset(options=...)` 当前主要开放以下参数：

| 路径 | 含义 |
| --- | --- |
| `options["reset_params"]["signeo"]` | 初始平衡相关参数，会影响初始电流和整体平衡状态 |
| `options["reset_params"]["bp"]` | 初始平衡相关参数，会影响等离子体位置和形状 |
| `options["reset_params"]["q0"]` | 初始平衡相关参数，会影响平衡与位形响应特性 |
| `options["reference_mode"]` | `hold` 或 `trajectory` |
| `options["reference"]` | 用户自定义 reference |

说明：

- 公开 reset 接口目前只暴露 `signeo`、`bp`、`q0`
- 底层场景由 `shot_id` 决定
- 这些参数主要用于切换或微调初始平衡状态

推荐将 `reset_params` 用作训练阶段的扰动增强。一个实用的起点是相对初始状态做如下随机扰动：

| 参数 | 推荐范围 | 训练作用 |
| --- | --- | --- |
| `signeo` | `±50%` | 增加初始电流和平衡状态变化，避免策略只适配单一初态 |
| `bp` | `±10%` | 增加位置和形状相关扰动，提高位形控制鲁棒性 |
| `q0` | `±10%` | 增加平衡响应差异，提高对隐藏场景的泛化能力 |

这些随机化本质上是一种面向控制任务的 domain randomization，有助于提升模型对初始偏差、建模误差和隐藏测试条件的适应能力。

### reference 训练建议

如果希望先做一个稳定的 baseline，建议优先围绕 `Ip` 和 `R` 做变轨迹训练：

- `Ip`：相对初始目标变化 `±100 kA`
- `R`：相对初始目标变化 `±3 cm`
- `Z`：基础阶段可先保持不变
- `lcfs_points`：基础阶段可先保持不变，后续再加入更复杂的边界跟踪任务

这样做的好处是先把最核心的电流和位置控制学稳，再逐步扩大到更复杂的位形控制目标。

## 7. 关键配置

`configs/env_default.yaml` 中最重要的配置项如下：

| 配置项 | 默认值 | 作用 |
| --- | --- | --- |
| `max_steps` | `100` | episode 最大步数 |
| `reference.mode` | `hold` | 默认 reference 模式 |
| `reference.reference_keys` | `["Ip", "R", "Z", "lcfs_points"]` | 当前 reference 覆盖的核心目标 |
| `predictor.host` | `127.0.0.1` | Docker 仿真器地址 |
| `predictor.port` | `5558` | Docker 仿真器端口 |
| `predictor.timeout` | `300.0` | socket 通信超时时间 |
| `predictor.shot_id` | `13844_500` | 默认预设环境模板 |

说明：

- `shot_id` 用于选择预设环境配置。每个 `shot` 固定了一组底层平衡文件和默认初始参数。
- 当前可选 `shot` 为 `13844_500`、`13844_600`、`15892_400`、`15892_500`
- 可以先固定一个 `shot` 跑通训练，再逐步加入 `reference` 变化和 `reset` 扰动

## 8. observation 快速理解

- `Ip`、`R`、`Z`、`lcfs_points` 是最直接的控制目标
- `reference_*` 给出当前步希望达到的目标值
- `I_PF` 提供"执行器当前处于什么状态"
- `Rmax`、`Rmin`、`aminor`、`kappa` 等提供"整体形状压缩信息"
- `Bm`、`Fx`、`rX`、`zX` 等提供"更细粒度的物理状态信息"

**快速上手建议**：
- 如果只想先快速做 baseline，可以先围绕核心目标量构造特征
- 如果希望提升泛化和稳定性，再逐步加入辅助观测量
